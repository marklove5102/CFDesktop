# .dockerignore

> 文档编写日期: 2026-03-20

## 使用办法 (Usage)

用于优化Docker构建上下文，排除不必要的文件，减少构建上下文大小并加快构建速度。

## Scripts详解

### 排除规则

| 类型 | 规则 |
|------|------|
| Git | `.git`, `.gitignore`, `.gitattributes` |
| 构建产物 | `out/`, `build/`, `*.o`, `*.a`, `*.so`, `*.dylib`, `*.dll`, `*.exe` |
| IDE | `.vscode/`, `.vs/`, `.idea/`, `*.user`, `*.suo` |
| 临时文件 | `*.tmp`, `*.log`, `*.swp`, `*~` |
| 操作系统 | `.DS_Store`, `Thumbs.db` |
| 文档 | `docs/`, `document/ci/`, `*.md`（保留`README.md`） |
| CI/CD | `.github/` |
| 依赖 | `node_modules/` |

### 特殊说明
- `!README.md` - 排除所有`*.md`文件后，重新包含`README.md`
- 构建上下文为项目根目录（`.`），即使Dockerfile位于`scripts/docker/`目录

### 作用
- 减少构建上下文大小
- 加快构建速度
- 避免敏感文件进入镜像
- 排除平台特定文件（如`.DS_Store`、`Thumbs.db`）
